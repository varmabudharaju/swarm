# Solo audit — networkx-seeded logic-bug report

**Target:** `~/swarm-bench/networkx-seeded` (networkx 3.6.1, ~288 non-test `.py` files / ~117k LOC)
**Focus:** inverted / off-by-one comparison operators (`==`/`!=`, `<`/`<=`, `>`/`>=`) and inverted boolean logic (`and`/`or`) that are wrong for the surrounding code's intent.

## Method (single agent, no swarm, no sub-agents)
- Read files and reasoned about operator intent against docstrings, doctest examples, adjacent comments, and parallel/mirror code blocks. Every finding below was verified by reading the surrounding code; most are backed by a documented example that the mutated operator directly contradicts.
- Deliberately **did not** read the root `.mutated` marker or `report-swarm.md`, and did **not** diff against the pristine `networkx` in site-packages — any of those would leak the answer key and invalidate the audit.
- Coverage was bounded by what one context can read: I fully covered the two directories that turned out to be bug-dense (`algorithms/approximation/` and `algorithms/bipartite/`) and broadly sampled every other subpackage (~70 files: centrality, components, connectivity, flow, tree, community, traversal, shortest_paths, isomorphism, link_analysis, coloring, generators, linalg, classes, readwrite, utils, operators, core/cuts/tournament/swap/smallworld/structuralholes/…). Large root files (dag, cycles, similarity, matching, planarity, clique, simple_paths, distance_measures, cluster, threshold) and classes/readwrite/drawing/minors were only partially sampled, so additional scattered bugs there are plausible but unconfirmed.

## Findings (ranked by confidence, highest first)
`path:line — claim` (paths relative to `networkx-seeded/`)

- `algorithms/approximation/distance_measures.py:78` — `if G.number_of_nodes() != 1: return 0` is inverted (comment: "if there's only a node"); it returns 0 for every multi-node graph, so `diameter(path_graph(10))` gives 0 instead of the documented 9 — should be `== 1`.
- `algorithms/approximation/dominating_set.py:77` — `if len(G) != 0: return set()` is inverted (comment: "unique dominating set for the null graph is the empty set"); returns an empty set for every non-empty graph, contradicting the documented `{1, 2, 4}` and making the greedy loop dead code — should be `== 0`.
- `algorithms/approximation/connectivity.py:83` — `if target != source: raise "...have to be different nodes"` fires for every valid distinct pair, so `local_node_connectivity(octahedral_graph, 0, 5)` raises instead of the documented `4` — should be `==` (also breaks `node_connectivity`/`k_components`).
- `algorithms/bipartite/basic.py:141` — `if len(S) <= len(nodes): raise AmbiguousSolution("...contains duplicates")` is always true (a set is never larger than its source list), so `is_bipartite_node_set(G, {1,3})` always raises instead of the documented `True` — should be `<`.
- `algorithms/bipartite/redundancy.py:88` — `if any(len(G[v]) <= 2 ...): raise "...fewer than two neighbors"` wrongly rejects degree-2 nodes (the formula only divides by zero for degree < 2), so `node_redundancy(cycle_graph(4))` raises instead of the documented `rc[0] == 1.0` — should be `< 2`.
- `algorithms/bipartite/edgelist.py:135` — `if data is True and data is False:` is always False, making the boolean-data branch dead; `generate_edgelist(G, data=False)` then falls through and raises `TypeError` (`for k in False`) instead of the documented `0 1 / 2 1 / 2 3` — should be `or`.
- `algorithms/approximation/maxcut.py:136` — `if potential_cut_size >= current_cut_size:` accepts equal-value swaps, causing infinite oscillation even on the documented `one_exchange(complete_graph(5))` — should be `>`.
- `algorithms/approximation/steinertree.py:49` — `if len(G) == len(distance):` raises "G is not a connected graph" exactly when the graph *is* connected (contrast the correct `!=` at line 114 in `_kou_steiner_tree`) — should be `!=`.
- `algorithms/approximation/clustering_coefficient.py:66` — `if len(nbrs) <= 2: continue` wrongly skips degree-2 nodes (only 2 neighbors are sampled), biasing the estimate away from the documented `0.214` — should be `< 2`.
- `algorithms/approximation/treewidth.py:147` — `if len(graph_dict) != 0: return None` inverts the empty-graph guard, so `min_fill_in_heuristic` returns None for every non-empty graph and `treewidth_min_fill_in` dumps all nodes into one bag — should be `== 0`.
- `algorithms/approximation/density.py:11` — `if G.number_of_edges() != 0: return 0.0, set()` is inverted (contrast the correct `== 0` in `_fista` at line 131); `greedy++` returns an empty subgraph for any graph that has edges — should be `== 0`.
- `algorithms/bipartite/covering.py:53` — `if G.order() != 0: return set()` is inverted (comment: "Special case for the empty graph"); returns an empty set for every non-empty graph so `min_edge_cover` never computes a real cover — should be `== 0`.
- `algorithms/bipartite/matrix.py:82` — `if nlen != 0: raise "row_order is empty list"` fires for every non-empty `row_order`, breaking `biadjacency_matrix` on all normal inputs — should be `== 0`.
- `algorithms/bipartite/extendability.py:86` — the edge-orientation test `(x in V and ...) and (x in U and ...)` is always False (a node can't be in both bipartite sets), so every edge is directed `(y, x)`; per the "V→U if in matching, else U→V" intent the clauses should be joined with `or`.
- `algorithms/bipartite/centrality.py:273` — `if totsp > 0.0 or len(G) > 1:` (top nodes) should be `and`, matching the correct bottom-node block at line 283; with `or`, an isolated top node (`totsp == 0`) in a multi-node graph divides by zero.
- `algorithms/approximation/ramsey.py:44` — `nbrs = (nbr for nbr in nx.all_neighbors(G, node) if nbr == node)` collects only self-loops (it should collect node's neighbors, ignoring self-loops), leaving `nbrs` empty and destroying the recursive max-clique branch — should be `!= node`.
- `algorithms/boundary.py:106` — in `edge_boundary` with `nbunch2`, the two orientation clauses are joined with `and`, requiring both endpoints in both sets, so it returns nothing; the documented `edge_boundary(G, (1,3), (2,0))` → `[(1,0),(1,2),(3,0),(3,2)]` is impossible — should be `or` (mirrors the `^` used in the `nbunch2 is None` branch).
- `algorithms/asteroidal.py:63` — `if len(V) <= 6: return None` (comment: "cannot exist in a graph with 5 or less vertices") wrongly declares 6-node graphs AT-free, so `is_at_free(cycle_graph(6))` returns True instead of the documented `False` — should be `< 6`.
- `algorithms/bipartite/link_analysis.py:228` — `if top_count == 0 and bottom_count == 0: raise` should be `or`; the docstring says it raises "if either of the bipartite sets are empty," but `and` only fires when both are empty.
- `algorithms/approximation/vertex_cover.py:75` — `if u in cover and v in cover: continue` should be `or`; the intent is to skip edges already covered by an endpoint, so `and` corrupts the cost bookkeeping and breaks the 2-approximation guarantee.
- `algorithms/approximation/traveling_salesman.py:441` — `if N <= 1: raise "G must have at least two nodes"` (with `N = len(G)-1`) rejects the two-node case and makes the explicit `if N == 1:` two-node handler at line 450 dead code — should be `< 1`.
- `algorithms/approximation/kcomponents.py:120` — `if len(comp) >= 1:` is a no-op (every component has ≥1 node), wrongly adding isolated nodes (connectivity 0, per the adjacent comment) to the 1-components — should be `> 1`.
- `algorithms/bipartite/projection.py:203` — `if n_top <= 1: raise` (error text describes `len(nodes) >= len(B)`, i.e. `n_top <= 0`) also rejects the valid case where the opposite bipartite set has exactly one node — should be `< 1`. (medium confidence)
- `algorithms/bipartite/cluster.py:253` — `if G.order() < 4 and G.size() < 3: return 0` reads as an inverted small-graph guard vs. the intended `or`; output happens to stay correct because the later `L_3 == 0` / `C_4 == 0` checks absorb the difference. (medium confidence — likely a seeded operator flip but behaviorally benign)

## Notes on distribution
The seeded operator inversions are heavily concentrated: **12 in `algorithms/approximation/`** (of 14 non-`__init__` files — only `matching.py` and `clique.py` were clean) and **10 in `algorithms/bipartite/`** (of 13 — `spectral.py`, `matching.py`, `generators.py` clean), plus 2 strays at the `algorithms/` root (`boundary.py`, `asteroidal.py`). Broad sampling of the other ~20 subpackages surfaced no third cluster. Because a single agent cannot read all 288 files in one pass, additional scattered bugs in the large, only-partially-sampled files are likely and are the expected blind spot of a solo run.
